# Contributing

Thanks for your interest in contributing to UrantiaHub. Every repo is MIT-licensed and we welcome PRs from anyone.

## Getting Started

1. Fork the repo you want to work on
2. Clone your fork locally
3. Create a feature branch (`git checkout -b my-feature`)
4. Make your changes
5. Push and open a PR against `main`

Each repo has a README with specific setup instructions (dependencies, dev server, etc.).

## What We're Looking For

- Bug fixes
- New features
- Documentation improvements
- Translations
- Performance improvements
- Test coverage

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Follow the existing code style in the repo you're contributing to
- Include a clear description of what your PR does and why

## Questions?

Open an issue in the relevant repo or email [kelson@urantia.dev](mailto:kelson@urantia.dev).
